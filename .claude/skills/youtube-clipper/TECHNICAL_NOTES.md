# Technical Pitfalls Record

This document records key technical issues and solutions encountered during the development of the YouTube Clipper Skill.

## 1. FFmpeg libass Support Issue

### Problem Description
Standard Homebrew FFmpeg does not include the libass library, which prevents the `subtitles` filter from burning subtitles.

### Error Message
```
No such filter: 'subtitles'
```

Or when checking filters:
```bash
$ ffmpeg -filters 2>&1 | grep subtitles
# No output
```

### Root Cause
- Homebrew's standard `ffmpeg` formula excludes certain non-core libraries to reduce package size
- libass is a subtitle rendering library used for the `subtitles` filter
- Without libass, FFmpeg cannot burn subtitles to video

### Solutions

#### macOS
Use `ffmpeg-full` instead of standard FFmpeg:

```bash
# Install ffmpeg-full
brew install ffmpeg-full

# Path (Apple Silicon)
/opt/homebrew/opt/ffmpeg-full/bin/ffmpeg

# Path (Intel)
/usr/local/opt/ffmpeg-full/bin/ffmpeg

# Verify libass support
/opt/homebrew/opt/ffmpeg-full/bin/ffmpeg -filters 2>&1 | grep subtitles
```

#### Other Systems
Compile FFmpeg from source, ensuring libass is included:

```bash
# Ubuntu/Debian
sudo apt-get install libass-dev
./configure --enable-libass
make
sudo make install

# Verify
ffmpeg -filters 2>&1 | grep subtitles
```

### Detection Logic
The detection logic implemented in `burn_subtitles.py`:

1. Prioritize checking the `ffmpeg-full` path (macOS)
2. Check if standard `ffmpeg` supports libass
3. If neither is met, show the installation guide

```python
def detect_ffmpeg_variant():
    # Check ffmpeg-full (macOS)
    if platform.system() == 'Darwin':
        full_path = '/opt/homebrew/opt/ffmpeg-full/bin/ffmpeg'
        if Path(full_path).exists():
            return {'type': 'full', 'path': full_path}

    # Check standard ffmpeg
    standard_path = shutil.which('ffmpeg')
    if standard_path:
        has_libass = check_libass_support(standard_path)
        return {'has_libass': has_libass}
```

## 2. File Path Space Issue

### Problem Description
The FFmpeg `subtitles` filter cannot correctly parse file paths containing spaces, even if quotes or escaping were used.

### Error Message
```
[Parsed_subtitles_0 @ 0x...] Unable to find '/path/with'
```

Notice the path is truncated at the space (`/path/with spaces` → `/path/with`).

### Example
```bash
# Failed attempts
ffmpeg -i video.mp4 -vf "subtitles='/path/with spaces/sub.srt'" output.mp4
ffmpeg -i video.mp4 -vf "subtitles=/path/with\ spaces/sub.srt" output.mp4
ffmpeg -i video.mp4 -vf subtitles="'/path/with spaces/sub.srt'" output.mp4

# All error with: Unable to find '/path/with'
```

### Root Cause
The FFmpeg `subtitles` filter has a path parsing bug that fails to properly handle:
- Spaces inside quotes
- Escaped spaces
- Mixed quotes

This is a known limitation of FFmpeg.

### Solution: Use a Temporary Directory

Core idea: Copy files to a **space-free** temporary directory, process them, and move them back.

```python
import tempfile
import shutil

def burn_subtitles(video_path, subtitle_path, output_path):
    # 1. Create temporary directory (guaranteed space-free path)
    temp_dir = tempfile.mkdtemp(prefix='youtube_clipper_')
    # e.g.: /tmp/youtube_clipper_abc123

    try:
        # 2. Copy files to temporary directory
        temp_video = os.path.join(temp_dir, 'video.mp4')
        temp_subtitle = os.path.join(temp_dir, 'subtitle.srt')
        shutil.copy(video_path, temp_video)
        shutil.copy(subtitle_path, temp_subtitle)

        # 3. Execute FFmpeg (space-free path)
        cmd = [
            'ffmpeg',
            '-i', temp_video,
            '-vf', f'subtitles={temp_subtitle}',
            temp_output
        ]
        subprocess.run(cmd, check=True)

        # 4. Move output file to target location
        shutil.move(temp_output, output_path)

    finally:
        # 5. Clean temporary directory
        shutil.rmtree(temp_dir, ignore_errors=True)
```

### Why does this work?
- The path generated by `tempfile.mkdtemp()` contains no spaces (usually `/tmp/xxx`)
- FFmpeg can correctly parse paths without spaces
- It's transparent to the user, who can provide input/output paths with spaces

### Other Attempted But Failed Solutions
❌ Double quotes: `subtitles="/path/with spaces/sub.srt"`
❌ Single quotes: `subtitles='/path/with spaces/sub.srt'`
❌ Escaped spaces: `subtitles=/path/with\ spaces/sub.srt`
❌ Mixed quotes: `subtitles="'/path/with spaces/sub.srt'"`
❌ FFmpeg `-filter_complex`: Still suffers the exact same issue

✅ **Only Working Solution**: Temporary directory approach

## 3. VTT to SRT Format Conversion

### Format Differences

| Feature | VTT | SRT |
|---------|-----|-----|
| Header | `WEBVTT` | None |
| Sequence | Optional | Mandatory (Starts from 1) |
| Time Separator | `.` (period) | `,` (comma) |
| Styling Info | Supported | Unsupported |

### Timestamp Formats

```
VTT:  00:00:00.000 --> 00:00:03.500
SRT:  00:00:00,000 --> 00:00:03,500
              ↑                  ↑
            comma              comma
```

### Conversion Implementation

```python

        r'(\d{2}:\d{2}:\d{2})\.(\d{3})',
        r'\1,\2',
        content
    )
```

### Caveats
- VTT might contain position info (`align:start position:0%`), which must be removed
- VTT might have multiline text; keep the multiline structure for SRT
- Timestamp format is strict: `HH:MM:SS,mmm` (must have hours)

## 4. Subtitle Timestamp Adjustment

### Problem Description
After clipping a video, subtitle timestamps need to be adjusted relative to the new start time.

### Example
Original video:
```
[00:02:00] Subtitle 1
[00:02:03] Subtitle 2
[00:02:06] Subtitle 3
```

After clipping 02:00-02:10, subtitles should become:
```
[00:00:00] Subtitle 1
[00:00:03] Subtitle 2
[00:00:06] Subtitle 3
```

### Implementation
```python
def adjust_subtitle_time(time_seconds, offset):
    """
    Adjust subtitle timestamp

    Args:
        time_seconds: Original time (seconds)
        offset: Offset parameter (seconds), i.e., clip start time

    Returns:
        float: Adjusted time
    """
    adjusted = time_seconds - offset
    return max(0.0, adjusted)  # Ensure no negative numbers
```

### Edge Case Handling
1. Subtitle entirely within time range: Keep
2. Subtitle entirely outside time range: Drop
3. Subtitle crosses interval boundaries:
   - Cap start time to 0 (if before range)
   - Cap end time to segment duration (if after range)

## 5. Batch Translation Optimization

### Problem
Translating line-by-line generates massive API calls, which is slow and costly.

### Data
- A 30-minute video: ~600 subtitle lines
- Line-by-line translation: 600 API calls
- Batch translation (20 lines/batch): 30 API calls
- **Saves 95% of API calls**

### Implementation Strategy

```python
def translate_batch(subtitles, batch_size=20):
    batches = []
    for i in range(0, len(subtitles), batch_size):
        batch = subtitles[i:i + batch_size]
        batches.append(batch)

    # Translate each batch together
    for batch in batches:
        # Merge into single text block
        batch_text = '\n'.join([sub['text'] for sub in batch])

        # Single API call to translate the whole batch
        translations = translate_text(batch_text)

        # Distribute translation results
        # ...
```

### Batch Size Selection
- **20 lines** is the sweet spot:
  - Below 20: Too many API calls
  - Above 30: Single prompt too long, degrades translation quality
  - 20-25: Optimal range

### Translation Quality Assurance
When batch translating:
1. Maintain contextual coherence
2. Keep each subtitle line separate (do not merge translated lines)
3. Return a JSON array to perfectly match original ordering

## 6. yt-dlp Best Practices

### Format Selection
```python
'format': 'bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080][ext=mp4]/best'
```

Explanation:
- `bestvideo[height<=1080]`: Max video height 1080p
- `[ext=mp4]`: Prefer mp4 format (better compatibility)
- `+bestaudio[ext=m4a]`: Combine with best audio
- `/best[height<=1080][ext=mp4]`: Fallback option
- `/best`: Final fallback

### Why restrict to 1080p?
1. File Size: 4K videos are huge (5-10GB)
2. Processing speed: FFmpeg takes too long
3. Target Platforms: Short video platforms mainly operate at 1080p or lower
4. Storage Management: Saves disk space

### Subtitle Download
```python
'writesubtitles': True,
'writeautomaticsub': True,  # Auto-generated subtitles as fallback
'subtitleslangs': ['en'],   # English subtitles
'subtitlesformat': 'vtt',   # VTT format
```

Priority:
1. Manual subtitles (if any)
2. Auto-generated subtitles (YouTube generated)

### Output Template
```python
'outtmpl': '%(title)s [%(id)s].%(ext)s'
```

Output example:
```
Anthropic's Amodei on AI [Ckt1cj0xjRM].mp4
Anthropic's Amodei on AI [Ckt1cj0xjRM].en.vtt
```

Why include the Video ID:
- Uniqueness: Prevents overwriting duplicates
- Traceability: Easy to track back to the original source

## 7. Bilingual Subtitle Stylizations

### SRT Format Bilingual
```srt
1
00:00:00,000 --> 00:00:03,500
This is English subtitle
这是中文字幕

2
00:00:03,500 --> 00:00:07,000
Another English line
另一行中文
```

### FFmpeg Burned Style
```bash
subtitles=subtitle.srt:force_style='FontSize=24,MarginV=30'
```

Parameters:
- `FontSize=24`: Font size (good for 1080p)
- `MarginV=30`: Bottom margin (vertical pixels)
- Default: White text with black outline

### Styling Recommendations

| Video Resolution | FontSize | MarginV |
|------------------|----------|---------|
| 720p             | 20       | 20      |
| 1080p            | 24       | 30      |
| 4K               | 48       | 60      |

## 10. Error Handling Best Practices

### Detailed Error Data
```python
try:
    result = subprocess.run(cmd, capture_output=True, text=True)
    if result.returncode != 0:
        print(f"❌ Command failed:")
        print(f"   Command: {' '.join(cmd)}")
        print(f"   Return code: {result.returncode}")
        print(f"   Error output:")
        print(result.stderr)
        raise RuntimeError("Command failed")
except Exception as e:
    print(f"❌ Error: {str(e)}")
    import traceback
    traceback.print_exc()
    sys.exit(1)
```

### User-Friendly Error Messages
```python
# ❌ Bad
raise Exception("FFmpeg failed")

# ✅ Good
raise RuntimeError(
    "FFmpeg does not support libass (subtitles filter). "
    "Please install ffmpeg-full: brew install ffmpeg-full"
)
```
